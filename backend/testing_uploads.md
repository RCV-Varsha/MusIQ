# MusIQ Backend: Testing Uploads & Mood Categorization

This guide explains how to test the stabilized upload pipeline using Postman or any HTTP client.

## API Endpoint: `POST /songs`

Upload multiple audio files with mood categorization.

### Postman Setup

1. **Method**: `POST`
2. **URL**: `http://localhost:3000/songs`
3. **Body**: Select `form-data`
4. **Keys**:
   - `songs`: Select **File** type and attach one or more `.mp3` or `.wav` files.
   - `detectedMood`: String (e.g., `happy`, `sad`, `neutral`).
   - `moods`: String OR Array.
     - String format: `feel-good, energetic, upbeat`
     - Array format: (Postman handles this by using multiple keys named `moods[]` or `moods`)

### Sample Payloads

#### Scenario A: AI Camera Upload (Single File)
- **songs**: `song1.mp3`
- **detectedMood**: `happy`
- **moods**: `happy, bright, energetic`

#### Scenario B: Bulk Categorization (Multiple Files)
- **songs**: `chill_vibes.mp3`, `evening_jazz.wav`
- **detectedMood**: `neutral`
- **moods**: `calm, focused, chill`

---

## API Endpoint: `GET /songs?mood=value`

Search for songs by mood. This searches across both `detectedMood` and the `moods` array.

### Examples
- `GET http://localhost:3000/songs?mood=happy`
- `GET http://localhost:3000/songs?mood=chill`

---

## Expected MongoDB Structure

Each song record in the `songs` collection will look like this:

```json
{
  "_id": "64b...",
  "title": "Summer Hits",
  "artist": "DJ MusIQ",
  "audio": "https://ik.imagekit.io/your_id/moodify/...",
  "detectedMood": "happy",
  "moods": ["happy", "bright", "energetic"],
  "createdAt": "2024-..."
}
```

## ImageKit URL Format
URLs are generated by the `storage.service.js` and stored in the `audio` field:
`https://ik.imagekit.io/<your_id>/moodify/<random_id>`

## Troubleshooting

1. **400 Error: No files uploaded**
   - Ensure the key in `form-data` is exactly `songs` (plural).
2. **400 Error: detectedMood is required**
   - Ensure you are sending `detectedMood` in the request body.
3. **500 Error: ImageKit upload failed**
   - Check `.env` for `IMAGEKIT_PRIVATE_KEY`, `IMAGEKIT_PUBLIC_KEY`, and `IMAGEKIT_URL_ENDPOINT`.
4. **Metadata Extraction Failure**
   - The backend will fallback to using the original filename as the title if metadata parsing fails.
